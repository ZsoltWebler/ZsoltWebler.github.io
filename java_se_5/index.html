    Öröklődés az objektum orientált programozásban

    Bevezetés

    Ebben a szekcióban az objektumorientált programozás egyik legfontosabb eszközével, az öröklődéssel fogunk megismerkedni. Az öröklődés segítségével a programunkban tovább tudjuk csökkenteni a kód duplikációt ezáltal a programunk flexibilisebb, rendezettebb lesz. Az öröklődés lehetővé teszi számunkra, hogy a meglévő funkcionalitást kibővítsük, illetve mi magunk is bővíthető kódot hozzunk létre, ezzel megalapozva a komplex, moduláris rendszerekben való munkát

    Az öröklődés alapjai

    Az öröklődés az objektum orientált elméletben azt jelenti, hogy az osztály alosztálya lehet egy másik osztálynak. Az alosztály megörökli az szülő osztály összes adattagját és metódusát. Az öröklődés tehát típus-altípus között van. Jármú -Állat, Állat-Háziállat-Kutya, stb...

    Osztályváltozók öröklése

    A java nyelvben egy osztálynak több alosztálya is lehet, de minden osztálynak csak egy ősosztálya van. Ez az úgynevezett többszörös öröklődés kiküszöbölésére van. Az öröklődés felfogható az őszosztály kibővítéseként is, ezért használja a java az extends kulcsszót az öröklődés során.

    class Szamitogep {
    String gyarto;
    String modell;
    int processzorMagok;
    int memoria;
    int hattertar;
}
 ...

class Laptop extends Szamitogep {
    int tomeg;
    int akkumulatorKapacitas;
}

    A fenti példában létrehozott Laptop a Szamitogep osztályt bővíti ki, ami azt jelenti, hogy ebben a példában, a Szamitogep az ősosztály és a Laptop a leszármazott, vagy gyerek osztály. A Laptop osztály megörökli, a Szamitogep osztály összes attribútumát.

    Metódusok öröklése.

    A kibővített osztályból nem csak osztály változókat, hanem metódusokat is öröklünk, tehát a lenti példában, miután hozzáadtunk egy új bekapcsol() metódust a Szamitogep, az automatikusan öröklődik a Laptop osztályba, ahol azt majd minden példányon meg lehet hívni, ugyan úgy, mint ha eleve a Laptop osztályban lett volna deklarálva.

    class Szamitogep {
 ...

    void bekapcsol() {
        System.out.println(" Betoltes ... ");
    }
}
 ...

class Laptop extends Szamitogep {

}

    Többszörös öröklődés

    Azért, mert egy osztály egy másik osztály alosztálya, attól még belőle további alosztályok is létrehozhatóak.

    class Szamitogep {
 ...
}
 ...

class Laptop extends Szamitogep {
 ...
}
 ...

class TwoInOne extends Laptop {
 ...
}

    A fenti példában létrehozott Laptop a Szamitogep osztályt öröklő, ami azt jelenti, hogy a saját tomeg es akkumulatorKapacitas változói mellett örökölte a Szamitogep osztály összes változóját is.

    Konstruktorok

    Ha a szülő osztály konstruktorának vannak paraméterei, akkor azokat a paramétereket az alosztályban ki kell elégíteni. Erre a super kulcs szót használjuk, aminek segítségével a szülő osztály konstruktorát hívjuk meg. A lenti példában a Teglalap osztály konstruktora két paramétert kér (magassag, szelesseg), igy a Negyzet alosztály konstruktorában kötelező ezt a két érteket megadni. Mivel egy négyzet minden oldala egyenlő hosszú lesz, a Negyzet konstruktorában csak egy számot kérünk be, es azt a számot adjuk meg a Teglalap konstruktor mindkét értekének.

    class Teglalap {
    int a, b;

    Teglalap(int a, int b) {
        this.a = a;
        this.b = b;
    }
}
 ...

class Negyzet {
    Negyzet(int a) {
        super(a, a);
    }
}

    A következő példában az alosztály nem kevesebb, hanem több értékét kér be mint a szülőosztály.

    class Jarmu {
    String modell;
    int ev;

    Jarmu(String modell, int ev) {
 ...
    }
}
 ...

class Auto {
    String rendszam;
    int ajtok;

    Auto(String modell, int ev, String rendszam, int ajtok) {
        super(gyarto, modell, ev);
        this.rendszam = rendszam;
        this.ajtok = ajtok;
    }
}

    Hozzáférés módosítók

    Az előző feladatsorban mar megismerkedtünk két hozzáférés módosítóval, a private-tal és a ”default”-tal. Mivel már tudjuk, mit jelent az, ha egy osztálynak egy másik osztály alosztálya, igy értelmezhetjük a másik két hozzáférés módosítót is. A protected es a public kulcsszavakkal a változókat es metódusokat elérhetővé tehetjük a csomagon kívülről is, azonban a protected a hozzáférést limitálni fogja az osztályunk alosztályaira. A private elemek továbbra is az osztályon belül lesznek csak elérhetők, a csomagon belül levő alosztályok sem férhetnek hozza.

    Kulcsszó Osztályon belül Csomagon belül bárhonnan Csomagon kívül alosztályból Csomagon kívül bárhonnan
    private Igen Nem Nem Nem
    (nincs) Igen Igen Nem Nem
    protected Igen Igen Igen Nem
    public Igen Igen Igen Igen

    Polimorfizmus

    A polimorfizmus az objektum orientált programozás alapelveinek egyike, jelentésé pedig ”több alakúság”. Akkor beszélünk polimorfizmusról, amikor egy típus példányai több alakot is felvehetnek. Ennek a legkézenfekvőbb példája az, amikor egy osztály alosztályai felül írjak a szülő osztály metódusait.

    class A {
    void M() {
        System.out.println("A ");
    }
}
 ...

class B extends A {
    void M() {
        System.out.println("B ");
    }
}
 ...

class C extends A {
    void M() {
        System.out.println("C ");
    }
}

    Ha az A osztályunkat kibővíti a B és C osztály, akkor egy A típussal deklarált változóba B és C típusokat is elhelyhetünk.

    A a=new A();
A a2=new B();
A a3=new C();

    Ez azt jelenti, hogy bar mind a három változónk A típusú, mivel más osztályok példányai, az M() metódus mind a három példányon mást fog kiírni a konzolra.

    @Override annotáció

    Metódusok felül írásakor erősen ajánlott az @Override annotáció használata, amivel egyértelműen jelezzük a Java-nak, hogy a szül osztályban levő metódus viselkedését fogjuk felül írni, de a ki- es bemeneteit nem. Ha azokat is felülijük, például egy új paramétert adunk hozza az alosztályban levő metódushoz, akkor az nem fogja felül írni a szülő osztály metódusát, hanem új metódusként adódik hozza az alosztályhoz. Egy @Override annotációval megjelölt metódusnál ebben az esetben mar futtatás elött figyelmeztetést fogunk kapni. Ha egy metódust az annotáció nélkül írunk felül, azt kitakarásnak vagy árnyékolásnak hívhatjuk.

    class C extends A {
    @Override
    void M() {
        System.out.println("C ");
    }
}

    Instaceof operátor

    Az instanceof operator segítségével ellenőrizheted, hogy egy változó egy bizonyos alosztályához tartozó objektumot tartalmaz-e. A lenti példában az előző A, B, C osztályokra hivatkozunk.

    A a=new B();
if(a instanceof B){
    System.out.println(" Ez egy B objektum");
}else if(a instanceof C){
    System.out.println(" Ez egy C objektum");
}

    Ha meggyőződtől róla, hogy az objektumod egy bizonyos alosztályhoz tartozik, azt biztonságban konvertálhatod az altípusba, ezzel helyben elérhetővé téve az alosztály sajátos metódusait, változóit.

    class A {
    // ures osztaly
}
 ...

class B extends A {
    void M() {
        System.out.println(" En  egy B vagyok !");
    }
}
 ...
 A a=new B();

 if(a instanceof B){
 B b=(B)a;
 b.M();
 }

    Final kulcsszó

    Ha meg szeretned akadályozni, hogy az osztályodnak alosztályai jöhessenek létre, vagy egy metódusod felül legyen írva egy alosztályban, egy osztály vagy metódust elláthatsz final kulcsszóval.

    final class Szamologep() {
    void osszead(int a, int b) {
        return a + b;
    }
}

     class Szamologep() {
    final void osszead(int a, int b) {
        return a + b;
    }
}

    Super kulcsszó

    A super kulcsszó nem csak konstruktoroknál használatos, általánosságban az osztály szülő osztályára mutat, a this -nek felel meg. Segítségével meg tudod hívni egy felüliért metódus eredeti verzióját a szülő osztályban. Az alábbi példában a Zsiraf osztály ki bővíti a bemutatkozását azzal, hogy megmondja a magasságát.

    class Allat {
    String nev;
    int kor;

    String bemutatkozas() {
        return "A nevem " + nev + " es " + kor + " eves vagyok ";
    }
}
 ...

class Zsiraf {
    int magassag;

    @Override
    String bemutatkozas() {
        return super.bemutatkozas() + " A magassagom "
                + magassag + " meter .";
    }
}

    Object osztály

    Ugyan ez számunkra nem látható, minden osztály automatikusan ki bővíti a java.lang.Object osztályt. Ez az osztály hierarchia csúcsa, ami meghatároz olyan metódusokat, amit minden Java objektumnak kötelező tartalmaznia, azonban ezeket az alosztályokkal felül írhatjuk. Figyelmesen tanulmányozd az Object osztályt es keresd meg az osztály Java dokumentációját az interneten. Az Object ősosztályból minden általunk létrehozott osztály örököl metódusokat, amiket mi szabadon felülírhatunk ilyenek például a toString() és az equals()

    toString() metódus

    A toString() metódus határozza meg, hogy egy objektumot hogyan reprezentálunk String alakban. Amikor fontos számunkra, hogy egy osztály példányait emberek szamara is érthetően jelenítsünk meg, olyankor ezt a metódust írjuk felül es String-kent adjuk vissza az objektumunk adatait, vagy bármit, amit a szituáció kíván. Az alapvető implementációja ennek a metódusnak csak az objektum osztályát és hash kódját fogja visszaadni, például Termek@1fee6fc;

    equals() metódus

    Az equals() metódus határozza meg, hogy mi alapján tekintjük egy osztály két példányát egyenlőnek. Fontos megjegyezni, hogy ennek a metódusnak a felülírása eseteben kötelező a hashCode() metódust is felülírni, hogy az osztályod konzisztenszen működjen hash alapú adatszerkezetekben, például HashMap(). Szerencsere ezt egyszerűen megtehetjük az Objects.hash() metódus használatával.

    public class Foglalas {
    String nev;
    int ora;

    @Override
    public boolean equals(Object o) {
 ...
    }

    @Override
    public int hashCode() {
        return Objects.hash(nev, ora);
    }
}

    Feladatok

    1. Feladat

    Mi történik, ha

    • megpróbálsz egy osztályból két másik osztályt ki bővíteni,
    • megpróbálod a Math osztályt kibővíteni,
    • egy szülő osztálynak statikus metódusai/változói vannak,
    • egy alosztályból nem elégíted ki a szülő osztály konstruktorának paramétereit,
    • egy tetszőleges T típusú tömbben T különböző alosztályainak objektumait tesszük,
    • egy tetszőleges T típussal deklarált változónak T szülő osztályának objektumát adjuk értekül,
    • egy tetszőleges T típus egyik alosztályával deklarált változójának a T osztály egy másik alosztályának objektumát adjuk értekül,
    • egy alosztályban létrehozott metódus vagy osztály változó neve megegyezik a szülő osztály egyik metódusával,
    • egy statikus metódust egy nem statikus metódussal árnyékolunk vagy fordítva,
    • két osztály egymást bővíti ki,
    • egy final kulcs szóval megjelölt osztályt vagy metódust megpróbálsz ki bővíteni vagy felül írni,
    • egy @Override annotációval megjelölt metódus neve/paraméterei/visszatérési érteke nem egyezik meg a szülő osztály egyik metódusával sem?

    2. Feladat

    Hozz létre egy tetszőleges osztályt és implementáld a toString() és equals() metódust.

    3. Feladat

    A következő feladatokban a lenti Termek osztallyal fogunk dolgozni.

    class Termek {
    private static int kovetkezoAzonosito = 100000;
    private int azonosito;
    private String megnevezes;
    private int ar;

    public Termek(String megnevezes, int ar) {
        this.azonosito = kovetkezoAzonosito++;
        this.megnevezes = megnevezes;
        this.ar = ar;
    }

    public int getAzonosito() {
        return azonosito;
    }

    public String getMegnevezes() {
        return megnevezes;
    }

    public int getAr() {
        return ar;
    }
}

    3.1. Feladat

    Készíts a Termek osztálynak egy Laptop alosztályt. Minden laptopról el szeretnénk tárolni a processzor típusát és a képátló hosszát. A Laptop alosztály a szülőosztálya mintájára készüljön, a változok hozzáférhetőségé, az osztály mutálhatósága, konstruktora stb. egyezzen meg vele.

    3.2. Feladat

    Készíts a Termek osztálynak egy Kamera alosztályt. Minden kameráról el szeretnénk tárolni a felbontást megapixelben. A Kamera alosztály a szülőosztálya mintájára készüljön, a változok hozzáférhetőségé, az osztály mutálhatósága, konstruktora stb. egyezzen meg vele.

    3.3. Feladat

    Készíts a Termek osztályba egy toString() metódust, amely a következő formátumú String-ben adja vissza a termék adatait.

    100014 BananaBook Pro 12 226000 Ft

    3.4. Feladat

    Egy Termek típusú tömbbe helyezz vegyesen Laptop és Kamera példányokat, majd hívd meg mindegyiken a toString() metódust egy ciklus segítségével.

    100000 BananaBook Pro 12 226000 Ft
100001 BananaBook Pro 15 265000 Ft
100002 Canikon D6000D 120000 Ft
100003 Canikon D6500D 165000 Ft

    3.5. Feladat

    Az instanceof operátorral módosítsd a ciklust úgy, hogy csak a Laptop osztály példányait jelenítse meg.

    3.6. Feladat

    Az instanceof operátorral módosítsd a ciklust úgy, hogy minden termékhez oda írja, hogy melyik kategóriába esik.

    100000 BananaBook Pro 12 226000 Ft Laptop
100001 BananaBook Pro 15 265000 Ft Laptop
100002 Canikon D6000D 120000 Ft Kamera
100003 Canikon D6500D 165000 Ft Kamera

    3.7. Feladat

    Írd felül a Laptop és Kamera osztály toString() metódusait úgy, hogy a ciklusod az alábbi eredményt produkálja.

    100000 BananaBook Pro 12 226000 Ft i5 6700K, 30 cm Laptop
100001 BananaBook Pro 15 265000 Ft amd 6300X, 24 cm Laptop
100002 Canikon D6000D 120000 Ft 24 MP cm Kamera
100003 Canikon D6500D 165000 Ft 36 MP Kamera

    3.8. Feladat

    Módosítsd úgy a Termek osztályt, hogy az equals() metódus két példányát akkor tekintse egyenlőnek, ha az azonosítóik megegyeznek.
