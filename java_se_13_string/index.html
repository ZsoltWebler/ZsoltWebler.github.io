    String

    Bevezetés

    A Java programozási nyelv egyik legfontosabb és leggyakrabban használt adattípusa a String. A String az egyszerű karakterláncokat, szavakat, mondatokat és akár hosszabb szövegeket is képes reprezentálni. Ebben a fejezetben áttekintjük a Java String osztályának alapvető jellemzőit, műveleteit, és bemutatjuk, hogy hogyan lehet hatékonyan használni a szövegek kezelésére.

    String osztály

    A String osztály egy kiemelten fontos adattípus a Java-ban, és a Java Standard könyvtár része. Az osztály immutábilis, vagyis egyszer létrehozott String objektumot nem lehet megváltoztatni. Ez azt jelenti, hogy a String objektumok értéke nem módosítható, csak új String objektumok hozhatók létre.

    Stringet több féle módon tudunk létrehozni:

            String szoveg1="Hello, World!"; // (1)
        String szoveg2=new String();     // (2)
        String szoveg3=new String("Java"); // (3)
    1. Értékadással történő inicializáció
    2. Üres sztring létrehozása konstuktor hívással
    3. Létrehozás literálból

    Mivel a String az egyik leggyakrabban használt osztály, így a nyelv az automatikus konstruktorhívás lehetőségét is felkínálja, azaz new hívás nélkül, egyszerű értékadással is inicializálható a String egy karaktersorozattal, vagy sztring konstanssal.

    Azonban a Stringek annyival többek már így is a karakter sorozatoknál, hogy értelmezve van rajtuk a "+" operátor, aminek hatására két sztring összefűzhető, és eredményül keletkezik egy harmadik String. Ez a "+" operátor igen érdekes olyan értelemben, hogy a Java a String kivételével nem teszi lehetővé ennek az operátornak a felüldefiniálását.

    String osztály metódusai

    A String osztály számos beépített metódust kínál a szövegek manipulálásához:

    length() - Ez a metódus visszaadja a String hosszát, azaz a benne található karakterek számát.

        String szoveg="Hello, World!";
        int hossz=szoveg.length(); // (1)
    1. A változó értéke 12

    charAt(index) - Ez a metódus visszaadja a megadott indexen található karaktert a String-ből. Az index 0-tól indul.

            String szoveg="Hello, World!";
        char karakter=szoveg.charAt(4); // (1)
    1. A karakter értéke 'o'

    substring(start, end) - Ez a metódus visszaad egy részszöveget a String-ből, a megadott kezdeti és záró index között. A kezdeti index beletartozik a részszövegbe, a záró index viszont már nem.

            String szoveg="Hello, World!";
        String reszszoveg=szoveg.substring(0,5); // (1)
    1. A változó értéke "Hello"

    toUpperCase() és toLowerCase() - Ezek a metódusok a szöveget átalakítják nagybetűssé vagy kisbetűssé

            String szoveg="Hello, World!";
        String nagybetus=szoveg.toUpperCase(); // (1)
        String kisbetus=szoveg.toLowerCase(); // (2)
    1. A változó értéke "HELLO, WORLD!"
    2. A változó értéke "hello, world!"

    equals(other) - Ez a metódus összehasonlítja két String-et tartalmilag. Visszaadja true-t, ha a két szöveg azonos, egyébként false-t.

            String szoveg1="Hello, World!";
        String szoveg2="Hello, World!";
        boolean azonos=szoveg1.equals(szoveg2); // (1)
    1. A változó értéke true

    indexOf(str) - Ez a metódus megtalálja az első előfordulást a megadott részszövegnek a String-ben, és visszaadja az első előfordulás indexét. Ha nem találja meg, -1-et ad vissza.

            String szoveg="Hello, World!";
        int index=szoveg.indexOf("World"); // (1)
    1. A változó értéke 7

    replace(oldStr, newStr) - Ez a metódus kicseréli az összes előfordulást a régi szövegből az új szövegre a String-ben.

            String szoveg="Hello, World!";
        String ujSzoveg=szoveg.replace("World","Java"); // (1)
    1. A változó értéke "Hello, Java!"

    split(delimiter) - Ez a metódus feldarabolja a String-et egy megadott elválasztó (delimiter) alapján, és egy tömbben tárolja az eredményeket.

            String szoveg="Alma,Körte,Szilva";
        String[]gyumolcsok=szoveg.split(","); // (1)
    1. A tömb tartalma: ["Alma", "Körte", "Szilva"]

    trim() - Ez a metódus eltávolítja a String elejéről és végéről az extra szóközöket (és más vezérlő karaktereket). Ez hasznos, ha a szöveget tisztítani szeretnénk a felesleges szóközöktől.

            String szoveg="   Java is awesome   ";
        String trimmed=szoveg.trim(); // (1)
    1. A változó értéke "Java is awesome"

    startsWith(prefix) és endsWith(suffix) - Ezek a metódusok ellenőrzik, hogy a String a megadott prefixszel kezdődik vagy a megadott suffixszel végződik.

            String szoveg="Hello, World!";
        boolean starts=szoveg.startsWith("Hello"); // (1)
        boolean ends=szoveg.endsWith("Java"); // (2)
    1. A változó értéke true
    2. A változó értéke false

    isEmpty() - Ez a metódus ellenőrzi, hogy a String üres-e (nincsenek benne karakterek). Visszaadja a true értéket, ha a String üres, egyébként false-t.

            String uresSzoveg="";
        boolean ures=uresSzoveg.isEmpty(); // (1)
    1. A változó értéke true

    isBlank() - Ez a metódus ellenőrzi, hogy a String üres vagy csak vezérlő karakterekből áll. Visszaadja a true értéket, ha a String üres, egyébként false-t.

            String uresSzoveg="  ";
        String uresTabulator="\t";
        boolean ures=uresSzoveg.isBlank(); // (1)
        boolean uresTab=uresTabulator.isBlank(); // (2)
    1. A változó értéke true
    2. A változó értéke true

    valueOf(data) - Ez a metódus egy String objektumot hoz létre az adat más típusából (pl. int, double) a String reprezentációval.

            int szam=42;
        String szoveg=String.valueOf(szam); // (1)
    1. A változó értéke "42"

    StringBuilder és StringBuffer

    Ahogy a korábbiakban tárgyaltuk, a String osztály objektumai immutable (módosíthatatlan) objektumok.

            String immutable="abc";
        immutable=immutable+"def";

    Mégha úgy is tűnik, hogy az String változónk értéke megváltozik, a valóságban az eredeti változónk referenciája felülírásra kerül egy új String objektummal, ami az "abcdef" karakterláncot tartalmazza. A String módosíthatatlanságának több oka is van leginkább teljesítmény és biztonsági okai, de ezekkel ebben a fejezetben nem foglalkozunk.

    Néha azonban nem praktikus, ha minden módosításunk után új String objektum jön létre. Ha gyakran módosítunk az adott karakterláncon felesleges, hogy mindig egy új String objektumot hozunk létre a módosításainkal. Ennek a problémának a megoldására készült a StringBuilder és StringBuffer osztály.

            StringBuffer sb=new StringBuffer("abc");
        sb.append("def");

    Az alábbi példa esetén az append() metódussal bővítjük az eredeti karakterláncunkat, nem jön létre új objektum, hanem a meglévő objektum bővül az új literállal.

    A StringBuffer szinkronizált, vagyis szálbiztos, StringBuilder kompatiblis a StringBuffer API-val, de nem garantálja a szálbiztonságot. Mivel nem szálbiztos implementáció, ezért valamivel gyorsabb a StringBuffer-nél, így, ha nem fontos a szálbiztonság használjuk ezt.

    String.format()

    A String osztály rendelkezésünkre bocsájt egy statikus format() metódust melynek segítségével a karakterláncunkat formázhatjuk. Nézzük az alábbi példát:

            String greetings=String.format("Hello %s !","World"); // (1)
    1. A kapott String a "Hello World !" lesz.

    A minta String statikus szöveget tartalmaz és egy vagy több formázási paramétert, amit be tudunk helyettesíteni különböző értékekkel. Ebben az esetben egy szimpla "%s" karaktert tartalmazott, ami a paraméterben kapott változót annak String reprezentációjával illeszti be a szövegbe.

    Általános szintaxis

    A formázás általános szintaxisa az alábbi:

    %[argument_index$][flags][width][.precision]conversion

    Az argument_index, flags, width, precision részek opcionálisan elhagyhatóak

    • argument_index egy szám, ami meghatározza, hogy hanyadik paraméterre hivatkozunk.
    • flags karakterek halmaza, amivel módosítani tudunk a formátumon
    • width segítségével megadhatjuk, hogy minimum hány karaktert foglaljon az adott behelyettesítés
    • precision segítségével megadhatjuk, hogy maximum hány karakter jelenjen meg
    • conversion az egyetlen kötelező rész, melynek segítségével megadhatjuk, hogy a változó milyen reprezentációban jelenjen meg a szövegben. A lehetséges karakterek halmaza attól függ, hogy milyen típusú a változó.
            String greetings_2=String.format("%2$s %1$s !","World","Hello"); // (1)
    1. A kapott String a "Hello World !" lesz.

    String formatter használata

    Nézzünk pár példát a String formatter használatára.

            String s=String.format("The correct answer is %s",false); // (1)
        s=String.format("The correct answer is %b",null); // (2)
        s=String.format("The correct answer is %B",true); // (3)
        s=String.format("The correct answer is %c",'a'); // (4)
        s=String.format("The correct answer is %c",null); // (5)
        s=String.format("The correct answer is %C",'b'); // (6)
        s=String.format("The valid unicode character: %c",0x0400); // (7)

        s=String.format("The number 25 in decimal = %d",25); // (8)
        s=String.format("The number 25 in octal = %o",25); // (9)
        s=String.format("The number 25 in hexadecimal = %x",25); // (10)
        s=String.format("The computerized scientific format of 10000.00 "+"= %e",10000.00); // (11)
        s=String.format("The decimal format of 10.019 = %f",10.019); // (12)
        s=String.format("First Line %nSecond Line"); // (13)
        s=String.format("Without left justified flag: %5d",25); // (14)
        s=String.format("With left justified flag: %-5d",25); // (15)

        s=String.format("Output of 25.09878 with Precision 2: %.2f",25.09878); // (16)
        s=String.format("Output of general conversion type with Precision 2: %.2b",true); // (17)
    1. A változó értéke "The correct answer is false"
    2. A változó értéke "The correct answer is false"
    3. A változó értéke "The correct answer is TRUE"
    4. A változó értéke "The correct answer is a"
    5. A változó értéke "The correct answer is null"
    6. A változó értéke "The correct answer is B"
    7. A változó értéke "The valid unicode character: Ѐ"
    8. A változó értéke "The number 25 in decimal = 25"
    9. A változó értéke "The number 25 in octal = 31"
    10. A változó értéke "The number 25 in hexadecimal = 19"
    11. A változó értéke "The computerized scientific format of 10000.00 = 1,000000e+04"
    12. A változó értéke "The decimal format of 10.019 = 10,019000"
    13. A változó értéke "First line"
      "Second line"
    14. A változó értéke "Without left justified flag: 25"
    15. A változó értéke "With left justified flag: 25 "
    16. A változó értéke "Output of 25.09878 with Precision 2: 25,10"
    17. A változó értéke "Output of general conversion type with Precision 2: tr"

    További példák és dokumentáció a https://docs.oracle.com/javase/8/docs/api/java/util/Formatter.html oldalon található.

    String joiner

    A StringJoiner osztály a Java 8-as verziójába került be a java.util package-be. Használatával Stringeket tudunk összefűzni előre definiált elválasztó karakter, prefix és suffix segítségéve.

            StringJoiner joiner=new StringJoiner(",","[","]"); //(1)
        joiner.add("Red").add("Green").add("Blue"); //(2)
        String joined=joiner.toString(); //(3)
    1. Létrehozzunk a StringJoiner objektumot a konstruktor (new StringJoiner(DELIMETER,PREFIX,SUFFIX)) hívással.
    2. Az add() metódus segítségével, hozzáadunk 3 karakterláncot, amit össze szeretnénk fűzni.
    3. Végül a toString() metódust meghívva elkészül a String objektumunk aminek értéke "[Red,Green,Blue]"

    Ahogy a példába láttuk az általános formája a StringJoiner konstruktorának a new StringJoiner(DELIMETER,PREFIX,SUFFIX), azonban a PREFFIX és SUFFIX tag opcionális.
