    Generikus típusok

    Bevezetés

    A Java 5 egyik kulcsfontosságú újítása a generikus típusok behozatala a Java nyelvbe. A generikus típusok lehetővé teszik a programozók számára, hogy általános, újrafelhasználható és típusbiztos kódot írjanak olyan helyzetekben, ahol különböző típusú adatokkal dolgoznak. Ez a fejezet mélyebben bemutatja a generikus típusok fogalmát, alkalmazásukat és előnyeiket.

    Generikus típusok használata

    A generikus típusok lehetővé teszik, hogy osztályokat, interfészeket és metódusokat írjunk, amelyek paraméterként egy általános típust fogadnak be, anélkül, hogy előre definiálnánk a konkrét típust. Ennek eredményeként a kód újrafelhasználható és rugalmasabb lesz, mivel ugyanazt a kódot többféle típussal használhatjuk.

    A generikus típusok használatával a programozók megőrizhetik a típusbiztonságot, és csökkenthetik a típushibák kockázatát. A fordítási időben történő típusellenőrzés biztosítja, hogy a kód csak az előre definiált típusokkal kompatibilis adatokkal működjön.

    Generikus típusokat azért használunk, hogy a programozó ne legyen kötve a konkrét típusokhoz, hanem egy általánosabb megoldást tudjon alkalmazni, amely többféle típussal is működik. Ez különösen hasznos adatszerkezetek (pl. listák, vermek, sorok) és algoritmusok esetén, ahol a kód egy általános megközelítést alkalmaz, függetlenül attól, hogy milyen típusú adatokkal dolgozik éppen.

    Generikus osztályok

    public class GenericList<T> { // (1)
    private Object[] elements; // (2)
    private int size; // (3)
    private static final int DEFAULT_CAPACITY = 10; // (4)

    public GenericList() {
        elements = new Object[DEFAULT_CAPACITY];
        size = 0;
    }

    public void add(T element) { // (5)
        if (size == elements.length) {
            resize();
        }
        elements[size++] = element;
    }

    public T get(int index) { // (6)
        if (index < 0 || index >= size) {
            throw new IndexOutOfBoundsException("Invalid index: " + index);
        }
        return (T) elements[index];
    }

    public int size() { // (7)
        return size;
    }

    private void resize() { // (8)
        int newCapacity = elements.length * 2;
        elements = java.util.Arrays.copyOf(elements, newCapacity);
    }

    public static void main(String[] args) {
        GenericList<String> stringList = new GenericList<>(); // (9)
        stringList.add("Apple");
        stringList.add("Banana");
        stringList.add("Orange");

        for (int i = 0; i < stringList.size(); i++) {
            System.out.println(stringList.get(i));
        }

        GenericList<Integer> intList = new GenericList<>(); // (10)
        intList.add(10);
        intList.add(20);
        intList.add(30);

        for (int i = 0; i < intList.size(); i++) {
            System.out.println(intList.get(i));
        }
    }
}
    1. Az osztályunkat kibővítjük a generikus <T> paraméterrel.
    2. Létrehozunk egy Object tömböt az elemeknek.
    3. A size változó tárolja a kollekciónk aktuális méretét.
    4. A DEFAULT_CAPACITY konstans tárolja a kollekciónk kezdő kapacitását.
    5. Létrehozunk egy add metódust ami a generikus T típusú objektumot fogad. És megpróbálja elhelyezni a kollekcióban. Ha nincs elég hely, akkor meghívja a resize metódust.
    6. Létrehozunk egy get metódust ami a generikus T típusú objektummal tér vissza. Ha olyan indexet kap, ami kívül esik a tömb intervallumán egy IndexOutOfBoundsException keletkezik.
    7. Visszatér a listánk aktuális méretével.
    8. Ha nincs elég hely a listánkban, akkor az elemek tárolására létrehoz egy új tömböt kétszer akkor mérettel, majd átmásolja az elemeket az új listába.
    9. Létrehozunk egy GenericList objektumot <String> típuskötéssel.
    10. Létrehozunk egy GenericList objektumot <Integer> típuskötéssel.

    A korábbi órák alkalmával többször hoztunk létre hasonló adatszerkezetet, hogy valamilyen osztály példányait el tudjuk tárolni egy dinamikus listában. A generikus típusok használatával lehetőségünk van egy általános megoldás elkészítésére, amit később bármilyen típusú objektum eltárolására tudunk használni.

    Generikus metódusok

    public class GenericMethod {

    public static <T> boolean equalTo(T t1, T t2) { // (1)
        return t1.equals(t2);
    }

    public static void main(String[] args) {
        boolean egyenlo_1 = GenericMethod.<String>equalTo("Alma", "Körte"); // (2)
        boolean egyenlo_2 = GenericMethod.<Integer>equalTo(6, 8); 
        boolean egyenlo_3 = GenericMethod.<Integer>equalTo(4, 6.4); // (3)
    }
}
    1. Az equalTo metódusunkat kibővítjük a generikus <T> paraméterrel, majd két generikus típusú objektumot várunk paraméterül.
    2. A metódus hívásakor le tudjuk kötni a generikus paramétert.
    3. Ha lekötjük a generikus paramétert, viszont olyan objektumokkal próbáljuk hívni a metódus, amik nem olyan típusúak fordítási hibát kapunk.

    A generikus osztályokon túl lehetőségünk van generikus metódusok létrehozására is. A generikus típus jelölést az osztályokhoz hasonlóan a <> gyémánt operátorral tudjuk jelölni a metódus szignatúrájában.

    Generikus név konvenciók

    A generikus típus jelölésére bármilyen karaktert használhatunk azonban a Javaban van néhény generikus típus név konvenció amihez, ha tartjuk magunkat megsegítjük a többi fejlesztő munkáját. Így ha tehetjük az alábbi esetekben használjuk a konvenció szerinti jelölést.

    • E - Elem (Element)
    • K - Kulcs (Key)
    • N - Szám (Number)
    • T - Típus (Type)
    • V - Érték (Value)
    • S, U stb. - További típusok

    Generikus típus korlátozás

    A generikus típus korlátozás lehetőséget nyújt arra, hogy megadjuk, hogy a generikus típusparaméter milyen típusokat fogadhat el vagy milyen feltételeknek kell megfelelnie. Ez segít pontosabban szabályozni a generikus kód viselkedését és hozzáadni bizonyos típusbiztonságot.

    public class GenericSorting {
    public static <T extends Comparable<T>> void bubbleSort(T[] array) { // (1)
        int n = array.length;
        for (int i = 0; i < n - 1; i++) { // (2)
            for (int j = 0; j < n - i - 1; j++) {
                if (array[j].compareTo(array[j + 1]) > 0) {
                    T temp = array[j];
                    array[j] = array[j + 1];
                    array[j + 1] = temp;
                }
            }
        }
    }

    public static void main(String[] args) {
        Integer[] intArray = {5, 2, 9, 1, 5, 6}; // (3)
        String[] strArray = {"Banana", "Apple", "Orange", "Grapes"}; // (4)

        System.out.println("Integer tömb rendezése:");
        bubbleSort(intArray); // (5)
        for (int num : intArray) {
            System.out.print(num + " ");
        }
        System.out.println();

        System.out.println("String tömb rendezése:");
        bubbleSort(strArray); // (6)
        for (String str : strArray) {
            System.out.print(str + " ");
        }
    }
}

    1. A bubbleSort metódusunkat kibővítjük a generikus <T extends Comparable<T>> paraméterrel. Az extends kulcsszó alkalmazásával előírjuk, hogy az a típus, amit átadunk a generikus metódusnak meg kell, hogy valósítsa a Comparable interfészt. Maga a Comparable interfész is generikus típusú, ezért átadjuk az ő generikus típusának az általunk definiált T típust.

    2. A buborékos rendezési algoritmus implementációja.

    3. Létrehozunk egy számokat tartalmazó tömböt.
    4. Létrehozunk egy szövegeket tartalmazó tömböt.
    5. Meghívjuk a rendezési algoritmusunkat a számokat tartalmazó tömbön.
    6. Meghívjuk a rendezési algoritmusunkat a szövegeket tartalmazó tömbön.

    Ebben a példában a bubbleSort metódus egy generikus típusú tömböt vesz át, amely implementálja a Comparable interfészt. Ez a metódus alkalmazza a buborékrendezést a tömb elemein, hogy növekvő sorrendbe rendezze őket. Így létrehoztunk egy általános rendezési algoritmust, amit bármilyen osztály objektumain tudunk használni, ha azok implementálják a Comparable interfészt.

    Comparable interfészről bővebben
    Comparable interface
    public interface Comparable<T> {
    int compareTo(T o);
}

    Az int compareTo(T o) metódus az interfész egyetlen absztrakt metódusa. Ezt a metódust az osztályoknak kell megvalósítaniuk, és vissza kell adniuk egy negatív, pozitív vagy nulla értéket attól függően, hogy az aktuális objektum kisebb, nagyobb vagy egyenlő-e a másik objektummal. A Java beépített osztályai közül sok támaszkodik az interfészre, hogy relációba állítson fel különböző objektumok között. Ha szeretnénk, hogy az osztályunk rendezhető legyen valamilyen logika mentén nekünk is implementálni kell a Comperable interfészt az osztályunkban.
