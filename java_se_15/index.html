    Java funkcionális programozás

    Bevezetés

    A Java, bár eredetileg egy objektumorientált nyelv, a funkcionális programozás számos elemét támogatja, különösen a Java 8 verzióval bevezetett új nyelvi elemek révén. Ezek a funkcionális programozási eszközök lehetővé teszik a fejlesztők számára, hogy a Java nyelvet a funkcionális paradigma irányábából használják. A funkcionális programozás ( functional programming) egy programozási módszertan, vagyis egyike a programozási paradigmáknak. A funkcionális programnyelvek a programozási feladatot egy függvény kiértékelésének tekintik.

    Anonim osztályok

    Az anonim osztályok segítségével tömörebbé tehetjük a kódunkat. Lehetővé teszik egy osztály deklarálását és példányosítását egyidejűleg. Olyan helyzetekben használunk anonim osztályokat, amikor valami eseti megoldásra van szükségünk és nem szeretnénk egy osztályt létrehozni a feladat elvégzésére.

    public interface ActionListener extends EventListener {

    public void actionPerformed(ActionEvent e);

}

    Ilyen eset például amikor egy gombnyomást szerettünk volna lekezelni ActionListener interfész segítségével. Ezt megtehettük egy anonim osztály segítségével, amit átadtunk a gombnak az addActionListener metódus segítségével.

    btnNewButton.addActionListener(new ActionListener(){
public void actionPerformed(ActionEvent e){
        //Gombnyomás hatása
        }
        });

    Ilyenkor ahelyett, hogy létrehoztunk volna egy osztályt, ami implementálja az ActionListener interfészt, helyben létrehoztunk egy anonim osztályt, és helyben megadtuk az actionPerformed metódus definícióját.

    Lambda kifejezések

    Az anonim osztályokkal kapcsolatos egyik fő probléma az, hogy ha az anonim osztály megvalósítása nagyon egyszerű, például egy olyan interfész, amely csak egy metódust tartalmaz, akkor az anonim osztályok szintaxisa feleslegesen bonyolultnak tűnik. A korábbi példánál maradva, mi egy gombnyomás eseményét szerettük volna lekezelni, megspóroltuk egy külön osztály létrehozását, azonban a feladathoz mérten így is feleslegesen bonyolult szerkezettel sikerült csak elérni a kívánt működést. A lambda kifejezéseket az ilyen problámák megoldására találták ki.

    Alakítsuk át a korábbi példánkat, hogy az anonim osztály helyett lambda kifejezéssel oldjuk meg gombnyomás kezelését.

    btnNewButton.addActionListener(e->{
        // Gombnyomás hatása
        });

    Ahogy a példán látjátok, ActionEvent e paramétert a lambda kifejezésünk ugyan úgy megkapja, de a felesleges "sallang" nélkül egyből megadhatjuk, hogy mi történjen a gombnyomás hatására. Az átalakítással a kódunk olvashatóbb és átláthatóbb lett.

    Lamda kifejezés szintaxisa

    Vegyük az alábbi Calculator osztályt.

    public class Calculator {

    interface IntegerMath {
        int operation(int a, int b);
    }

    public int operateBinary(int a, int b, IntegerMath op) {
        return op.operation(a, b);
    }

    public static void main(String... args) {

        Calculator myApp = new Calculator();
        IntegerMath addition = (a, b) -> a + b;
        IntegerMath subtraction = (a, b) -> a - b;
        System.out.println("40 + 2 = " +
                myApp.operateBinary(40, 2, addition));
        System.out.println("20 - 10 = " +
                myApp.operateBinary(20, 10, subtraction));
    }
}

    A lambda kifejezéseket csak olyan esetben használhatjuk, ha az interfészünk összesen egy absztrakt metódust definiál, de default és static metódusból bármennyit tartalmazhat. Hozzuk létre az IntegerMath interfészünket és definiáljunk benne egy operation metódust. Az operation lesz a metódusunk amire majd a lambda kifejezést szeretnénk illeszteni. A Calculator osztályba hozzunk létre egy operateBinary metódust, ami két integer argumentumot fogad és elvégzi rajta az operation metódust.

    A main metódusban hozzunk létre két műveletet az egyiket az összeadás műveletével a másikat pedig a kivonáséval, majd próbáljuk ki őket.

    Funkcionális interfészek

    A @FunctionalInterface annotáció hasznos, hogy fordítási időben ellenőrizni tudjuk, hogy egy interfésznek csak egy absztrakt metódusa van. Mivel csak olyan interfészeket használhatunk lambda kifejezésre amiknek csak egy absztrakt metódusa van, ha az interfészeinket ellátjuk ezzel az annotációval, akkor fordítási időben meggyőzödhetünk erről, továbbá jelezni tudjuk a fejlesztő kollégáinknak is, hogy ne adjanak hozzá új metódust az interfészhez, mert mi azt lambda kifejezésekkel használjuk.

    @FunctionalInterface
interface IntegerMath {
    int operation(int a, int b);
}

    Method reference

    A metódus hivatkozás egy speciális típusa a lambda kifejezéseknek. Gyakran hozunk létre lambda kifejezést, hogy meghívjunk egy már létező metódust. Az ilyen egyszerű lambda kifejezések kiváltására hozták létre a metódus hivatkozásokat.

    Négy lehetséges típusa van a metódus hivatkozásoknak:

    • Statikus metódus
    • Adott objektum metódusa
    • Tetszőleges objektum bizonyos metódusa
    • Konstruktor

    Hivatkozás egy statikus metódusra

    Lehetőségünk van a lambda kifejezést kiváltani egy olyan statikus metódussal, aminek a paraméterlistája megegyezik a lambda kifejezés paraméterlistájával. Vegyük az alábbi példát, ami kiírja a String lista tartalmát a konzolra és alakítsuk át, hogy method reference segítségével oldjuk meg a feladatot.

    public class StaticMethod {

public static void main (String... args){
    List<String> messages = Arrays.asList("hello", "backend", "csoport!");
    messages.forEach(word -> System.out.println(word));
    }
}

    public class StaticMethod {

public static void main (String... args){
    List<String> messages = Arrays.asList("hello", "backend", "csoport!");
    messages.forEach(System.out::println);
    }
}

    Hivatkozás egy objektum metódusára

    Vegyük az alábbi Bicycle osztályt és a hozzá készített BicycleComparator osztályt, amivel két bicikli példányt tudunk összehasonlítani.

    public class Bicycle {

    private String brand;
    private Integer frameSize;
    // standard constructor, getters and setters
}

public class BicycleComparator implements Comparator {

    @Override
    public int compare(Bicycle a, Bicycle b) {
        return a.getFrameSize().compareTo(b.getFrameSize());
    }

}

    public class Bicycle {
    public static void main(String[] args) {
        BicycleComparator bikeFrameSizeComparator = new BicycleComparator();
        List<Bicycle> bicycleList = Arrays.asList(
                new Bicycle("MTK", 28),
                new Bicycle("TREK", 26),
                new Bicycle("FUJI", 28),
                new Bicycle("KONA", 26)
        );

        bicycleList.stream().sorted((a, b) -> bikeFrameSizeComparator.compare(a, b));
    }
}

    public class Bicycle {
    public static void main(String[] args) {
        BicycleComparator bikeFrameSizeComparator = new BicycleComparator();
        List<Bicycle> bicycleList = Arrays.asList(
                new Bicycle("MTK", 28),
                new Bicycle("TREK", 26),
                new Bicycle("FUJI", 28),
                new Bicycle("KONA", 26)
        );

        bicycleList.stream().sorted(bikeFrameSizeComparator::compare);
    }
}

    Hivatkozás egy objektum metódusára, anélkül, hogy példányosítanánk azt

    Ez a metódushivatkozás nagyon hasonlít az előzre, de ott létre kellett hoznunk, egy BicycleComparator példányt, hogy használni tudjuk. Itt az Integer osztály metódusát tudjuk használni.

    public class MethodReference {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(5, 3, 50, 24, 40, 2, 9, 18);
        numbers.stream().sorted((a, b) -> a.compareTo(b));
    }
}

    public class MethodReference {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(5, 3, 50, 24, 40, 2, 9, 18);
        numbers.stream().sorted(Integer::compareTo);
    }
}

    Hivatkozás egy objektum konstruktorára

    Egészítsük ki a korábbi Bicycle osztályt egy konstruktorral ami egy brand paramétert fogad. Ha létrehozunk egy listát ami a gyártókat tartalmazza egy egyszerű kifejezéssel ebből a String listából Bicycle objektumokat tudunk létrehozni.

    class Bicycle {

    //...
    public Bicycle(String brand) {
        this.brand = brand;
        this.frameSize = 0;
    }
    //...
}

        public class Bicycle {
        public static void main(String[] args) {
            List<String> bikeBrands = Arrays.asList("Giant", "Scott", "Trek", "GT");
            bikeBrands.stream().map(brand -> new Bicycle(brand)).toList();
    }
}

        public class Bicycle {
        public static void main(String[] args) {
            List<String> bikeBrands = Arrays.asList("Giant", "Scott", "Trek", "GT");
            bikeBrands.stream().map(Bicycle::new).toList();
    }
}

    Streams

    Az egyik leghasznosabb újdonság a Java 8 ban a Stream-ek voltak. Nem összekeverendő a Java I/O streamekkel (FileInputStream, stb).

    A Java Stream API egy hatékony módszer a kollekciók (például listák, tömbök stb.) eleminek feldolgozására. A Stream egy olyan objektum, amely lehetővé teszi funkcionális műveletek sorozatának végrehajtását az adatelemeken, mint például szűrés, leképezés és rendezés. A Stream API használata lehetővé teszi a kód olvashatóságának és karbantarthatóságának javítását, valamint az adatfeldolgozás hatékonyságát.

    A stream nem tárol adatot, nem egy adatstruktúra és soha nem módosítja az eredeti adat forrást.

    Stream létrehozása

    A Stream objektumot egy meglévő adatszerkezetből lehet létrehozni. Nézzünk meg pár leggyakrabban használt Stream létrehozási módszert.

    Stream kollekcióból

    A leggyakoribb módja a Stream létrehozásának egy meglévő kollekcióból (pl. List, Set, Map) való átalakítás. Ehhez a kollekció a stream() metódusát használjuk.

            List<String> myList=Arrays.asList("a","b","c");
        Stream<String> stream=myList.stream();

    Stream tömbből

    Tömbből is könnyen lehet Stream-et készíteni a Arrays.stream() metódussal.

            String[]array={"apple","banana","cherry"};
        Stream<String> stream=Arrays.stream(array);

    Iterált stream

    A Stream.iterate() metódus segítségével egy kezdeti értékből kiindulva sorozatot generálhatunk egy megadott szabály alapján.

            Stream<Integer> numbers=Stream.iterate(0,n->n+1);

    Stream műveletek

    AStream API számos műveletet kínál, amelyek lehetővé teszik az adatok kiválasztását, transzformálását és szűrését. Ezek a műveletek funkcionális programozási elveken alapulnak, és lehetővé teszik az adatfeldolgozás kifejező és hatékony kódolását. A Stream műveletek között megkülönböztethetünk köztes (intermediate) és terminális ( terminal) műveleteket.

    Intermediate műveletek

    Az intermediate stream műveletek közé tartoznak azok a műveletek, amelyek egy meglévő Stream-et módosítanak, átalakítanak vagy szűrnek, és új Stream-et eredményeznek anélkül, hogy a végleges eredményt kiszámítanák. Ezek a műveletek "köztes" műveletek, mert további műveletekkel kombinálhatók a kívánt eredmény eléréséhez. Az intermediate műveletek nem hajtanak végre tényleges adatfeldolgozást, csak egy új Stream-et hoznak létre az eredeti Stream alapján.

    Filter

    A filter metódus segítségével szűrhetjük az elemeket egy Stream-ben egy adott feltétel alapján. Csak azok az elemek kerülnek be a kimeneti Stream-be, amelyek a feltételnek megfelelnek.

            List<Integer> numbers=Arrays.asList(1,2,3,4,5,6);
        Stream<Integer> evenNumbers=numbers.stream().filter(n->n%2==0); // (1)
    1. A kimeneti Stream csak a páros elemeket tartalmazza.
    Map

    A map metódus lehetővé teszi az elemek átalakítását egy Stream-ben egy megadott függvény segítségével.

            List<String> words=Arrays.asList("apple","banana","cherry");
        Stream<Integer> wordLengths=words.stream().map(String::length); // (1)
    1. A kimeneti Stream a szavak hosszúságát tartalmazza.
    Flatmap

    A flatMap metódus lehetővé teszi egy új Stream létrehozását az eredeti Stream elemeiből egy megadott függvény által létrehozott Stream-ek összefésülésével.

            List<List<Integer>>nestedLists=Arrays.asList(
        Arrays.asList(1,2,3),
        Arrays.asList(4,5,6),
        Arrays.asList(7,8,9)
        );
        Stream<Integer> flattenedStream=nestedLists.stream().flatMap(List::stream); // (1)
    1. Ebben az esetben a flattenedStream tartalmazza az összes számot az al-Stream-ek összefésülésével.
    Distinct

    A distinct metódus egy új Stream-et hoz létre, amely csak az egyedi (ismétlédések nélküli) elemeket tartalmazza az eredeti Stream-ből.

            List<Integer> numbers=Arrays.asList(1,2,2,3,3,3,4,4,4,4);
        Stream<Integer> distinctNumbers=numbers.stream().distinct(); // (1)
    1. A kimeneti Stream csak a unique értékeket tartalmazza. (1, 2, 3, 4)
    Sorted

    A sorted metódus sorba rendezheti a Stream elemeit. Alapértelmezés szerint növekvő sorrendben rendez.

            List<String> names=Arrays.asList("John","Alice","Bob","Eve");
        Stream<String> sortedNames=names.stream().sorted();
    Peek

    A peek metódus lehetővé teszi, hogy egy függvényt hajtsunk végre minden elemen a Stream-ben, miközben a Stream tovább folyik. Ez gyakran hasznos a hibakereséshez vagy a logoláshoz, anélkül hogy befolyásolná az eredmény Stream-et.

            List<Integer> numbers=Arrays.asList(1,2,3,4,5);
        List<Integer> squaredNumbers=numbers.stream()
        .peek(n->System.out.println("Szám: "+n))
        .map(n->n*n)
        .collect(Collectors.toList());

    Az alábbi példa, először kiírja az összes számot, majd négyzetre emeli a számokat és a négyzetre emelt számok listájával tér vissza.

    Limit és Skip

    A limit és skip metódusok segítségével lehetőség van a Stream elemeinek korlátozására vagy kihagyására. A limit csak az első n darab elemet tartalmazza, míg a skip az első n elemet kihagyja a Stream-ből.

            List<Integer> numbers=Arrays.asList(1,2,3,4,5,6);
        Stream<Integer> limitedStream=numbers.stream().limit(3); // (1)
        Stream<Integer> skippedStream=numbers.stream().skip(2);   // (2)
    1. Csak az első 3 szám
    2. Az első 2 szám kihagyása

    Terminal műveletek

    Amikor egy terminális műveletet alkalmazunk egy Stream-re, a Stream lezárul, és nem hajt végre további köztes vagy terminális műveleteket.

    foreach

    A forEach művelet lehetővé teszi, hogy egy függvényt hajtsunk végre a Stream minden elemén. Ez hasznos, ha szeretnénk minden elemet feldolgozni a Stream-ből, például kiírni egy kollekció elemeit a konzolra.

            List<String> names=Arrays.asList("John","Alice","Bob");
        names.stream().forEach(name->System.out.println("Név: "+name));
    toArray

    A toArray művelet lehetővé teszi a Stream elemeinek tömbbe konvertálását. A kimeneti tömb típusát a toArray() metódus paraméterként lehet megadni.

            List<String> names=Arrays.asList("John","Alice","Bob");
        String[]nameArray=names.stream().toArray(String[]::new);
    anyMatch

    Az anyMatch metódus megnézi, hogy legalább egy elem a Stream-ben megfelel-e a megadott feltételnek. Ha talál egy megfelelő elemet, true-t ad vissza, egyébként false-t.

            List<Integer> numbers=Arrays.asList(1,2,3,4,5);
        boolean anyEven=numbers.stream().anyMatch(n->n%2==0);
    allMatch

    Az allMatch metódus ellenőrzi, hogy minden elem a Stream-ben megfelel-e a megadott feltételnek. Ha minden elem megfelel a feltételnek, true-t ad vissza, egyébként false-t.

            List<Integer> numbers=Arrays.asList(1,2,3,4,5);
        boolean allEven=numbers.stream().allMatch(n->n%2==0);
    noneMatch

    A noneMatch metódus megnézi, hogy egyetlen elem sem felel-e meg a megadott feltételnek. Ha nincs megfelelő elem, true-t ad vissza, egyébként false-t.

            List<Integer> numbers=Arrays.asList(1,3,5,7,9);
        boolean noneEven=numbers.stream().noneMatch(n->n%2==0);
    count

    A count metódus visszaadja a Stream elemeinek számát.

            List<Integer> numbers=Arrays.asList(1,2,3,4,5);
        long count=numbers.stream().count();
    findAny

    A findAny metódus bármely elemet visszaadja a Stream-ből. Ez hasznos, ha csak egy elemre van szükségünk, és nincs jelentősége annak, hogy pontosan melyik elemet kapjuk vissza.

            List<String> fruits=Arrays.asList("apple","banana","cherry");
        Optional<String> anyFruit=fruits.stream().findAny();
    findFirst

    A findFirst metódus visszaadja az első elemet a Stream-ből.

            List<Integer> numbers=Arrays.asList(1,2,3,4,5);
        Optional<Integer> firstNumber=numbers.stream().findFirst();
    min és max

    Ezek a metódusok a Stream legkisebb vagy legnagyobb elemét adják vissza a megadott összehasonlítás alapján.

            List<Integer> numbers=Arrays.asList(5,2,8,1,6);
        Optional<Integer> minNumber=numbers.stream().min(Integer::compareTo);
        Optional<Integer> maxNumber=numbers.stream().max(Integer::compareTo);
    reduce

    A reduce metódus lehetővé teszi a Stream elemeinek összevonását egyetlen értékbe a megadott összefűző művelet segítségével.

            List<Integer> numbers=Arrays.asList(1,2,3,4,5);
        int sum=numbers.stream().reduce(0,(a,b)->a+b);
    collect

    A collect művelet lehetővé teszi az összes elemet egy gyűjtő objektumba gyűjteni, például egy List-be vagy Map-be. A Collector objektumot a Collectors osztály statikus metódusaival lehet létrehozni.

    Gyűjtés List-be
            List<String> names=Arrays.asList("John","Alice","Bob");
        List<String> collectedNames=names.stream().collect(Collectors.toList());
    Gyűjtés Set-be
            List<String> names=Arrays.asList("John","Alice","Bob","Alice");
        Set<String> collectedNames=names.stream().collect(Collectors.toSet());
    Gyűjtés Map-be
            List<String> names=Arrays.asList("John","Alice","Bob");
        Map<Integer, String> nameMap=names.stream().collect(Collectors.toMap(String::length,name->name));
    Szöveg összefűzése
            List<String> fruits=Arrays.asList("apple","banana","cherry");
        String concatenatedFruits=fruits.stream().collect(Collectors.joining(", "));
    Csoportosítás kulcs alapján
            List<String> names=Arrays.asList("John","Alice","Bob","Anna");
        Map<Character, List<String>>groupedNames=names.stream()
        .collect(Collectors.groupingBy(name->name.charAt(0)));
    Partícionálás
            List<Integer> numbers=Arrays.asList(1,2,3,4,5,6,7,8,9);
        Map<Boolean, List<Integer>>evenOddMap=numbers.stream()
        .collect(Collectors.partitioningBy(n->n%2==0));

    Optional

    Optional egy olyan osztály a Java-ban, amely lehetővé teszi, hogy kezeljük az érték nélküli vagy null értékeket anélkül, hogy null ellenőrzéseket kellene végezni. Az Optional osztály először a Java 8-ban jelent meg, és azóta használják a kódbázisokban az értékek biztonságosabb és olvashatóbb kezelése érdekében.

    Optional létrehozása

            Optional<String> emptyOptional=Optional.empty(); // (1)
        Optional<String> nonEmptyOptional=Optional.of("Hello, Optional!"); // (2)
        Optional<String> nullOptional=Optional.ofNullable(null); // (3)
    1. Üres Optional létrehozása
    2. Optional létrehozása értékből
    3. Optional létrehozása null objektumból

    Optional értékénének ellőrzése - isPresent

            Optional<String> optional = ...; // valamilyen Optional példány
        if (optional.isPresent()) {
            String value = optional.get();
            System.out.println("Az érték: " + value);
        } else {
            System.out.println("Nincs érték.");
        }

    Optional értékének használata - ifPresent

            Optional<String> optional = ...; // valamilyen Optional példány
        optional.ifPresent(value -> System.out.println("Az érték: " + value));

    Optional alapértelmezett értéke

            Optional<String> optional = ...; // valamilyen Optional példány
        String value = optional.orElse("Nincs érték");

    Kivétel dobása, ha nincs jelen érték

            Optional<String> optional = ...; // valamilyen Optional példány
        String value = optional.orElseThrow(() -> new RuntimeException("Nincs érték"));
