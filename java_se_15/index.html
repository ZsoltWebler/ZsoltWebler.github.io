    Java funkcionális programozás

    Bevezetés

    A Java, bár eredetileg egy objektumorientált nyelv, a funkcionális programozás számos elemét támogatja, különösen a Java 8 verzióval bevezetett új nyelvi elemek révén. Ezek a funkcionális programozási eszközök lehetővé teszik a fejlesztők számára, hogy a Java nyelvet a funkcionális paradigma irányábából használják. A funkcionális programozás ( functional programming) egy programozási módszertan, vagyis egyike a programozási paradigmáknak. A funkcionális programnyelvek a programozási feladatot egy függvény kiértékelésének tekintik.

    Anonim osztályok

    Az anonim osztályok segítségével tömörebbé tehetjük a kódunkat. Lehetővé teszik egy osztály deklarálását és példányosítását egyidejűleg. Olyan helyzetekben használunk anonim osztályokat, amikor valami eseti megoldásra van szükségünk és nem szeretnénk egy osztályt létrehozni a feladat elvégzésére.

    public interface ActionListener extends EventListener {

    public void actionPerformed(ActionEvent e);

}

    Ilyen eset például amikor egy gombnyomást szerettünk volna lekezelni ActionListener interfész segítségével. Ezt megtehettük egy anonim osztály segítségével, amit átadtunk a gombnak az addActionListener metódus segítségével.

    btnNewButton.addActionListener(new ActionListener(){
public void actionPerformed(ActionEvent e){
        //Gombnyomás hatása
        }
        });

    Ilyenkor ahelyett, hogy létrehoztunk volna egy osztályt, ami implementálja az ActionListener interfészt, helyben létrehoztunk egy anonim osztályt, és helyben megadtuk az actionPerformed metódus definícióját.

    Lambda kifejezések

    Az anonim osztályokkal kapcsolatos egyik fő probléma az, hogy ha az anonim osztály megvalósítása nagyon egyszerű, például egy olyan interfész, amely csak egy metódust tartalmaz, akkor az anonim osztályok szintaxisa feleslegesen bonyolultnak tűnik. A korábbi példánál maradva, mi egy gombnyomás eseményét szerettük volna lekezelni, megspóroltuk egy külön osztály létrehozását, azonban a feladathoz mérten így is feleslegesen bonyolult szerkezettel sikerült csak elérni a kívánt működést. A lambda kifejezéseket az ilyen problámák megoldására találták ki.

    Alakítsuk át a korábbi példánkat, hogy az anonim osztály helyett lambda kifejezéssel oldjuk meg gombnyomás kezelését.

    btnNewButton.addActionListener(e->{
        // Gombnyomás hatása
        });

    Ahogy a példán látjátok, ActionEvent e paramétert a lambda kifejezésünk ugyan úgy megkapja, de a felesleges "sallang" nélkül egyből megadhatjuk, hogy mi történjen a gombnyomás hatására. Az átalakítással a kódunk olvashatóbb és átláthatóbb lett.

    Lamda kifejezés szintaxisa

    Vegyük az alábbi Calculator osztályt.

    public class Calculator {

    interface IntegerMath {
        int operation(int a, int b);
    }

    public int operateBinary(int a, int b, IntegerMath op) {
        return op.operation(a, b);
    }

    public static void main(String... args) {

        Calculator myApp = new Calculator();
        IntegerMath addition = (a, b) -> a + b;
        IntegerMath subtraction = (a, b) -> a - b;
        System.out.println("40 + 2 = " +
                myApp.operateBinary(40, 2, addition));
        System.out.println("20 - 10 = " +
                myApp.operateBinary(20, 10, subtraction));
    }
}

    A lambda kifejezéseket csak olyan esetben használhatjuk, ha az interfészünk összesen egy absztrakt metódust definiál, de default és static metódusból bármennyit tartalmazhat. Hozzuk létre az IntegerMath interfészünket és definiáljunk benne egy operation metódust. Az operation lesz a metódusunk amire majd a lambda kifejezést szeretnénk illeszteni. A Calculator osztályba hozzunk létre egy operateBinary metódust, ami két integer argumentumot fogad és elvégzi rajta az operation metódust.

    A main metódusban hozzunk létre két műveletet az egyiket az összeadás műveletével a másikat pedig a kivonáséval, majd próbáljuk ki őket.

    Funkcionális interfészek

    A @FunctionalInterface annotáció hasznos, hogy fordítási időben ellenőrizni tudjuk, hogy egy interfésznek csak egy absztrakt metódusa van. Mivel csak olyan interfészeket használhatunk lambda kifejezésre amiknek csak egy absztrakt metódusa van, ha az interfészeinket ellátjuk ezzel az annotációval, akkor fordítási időben meggyőzödhetünk erről, továbbá jelezni tudjuk a fejlesztő kollégáinknak is, hogy ne adjanak hozzá új metódust az interfészhez, mert mi azt lambda kifejezésekkel használjuk.

    @FunctionalInterface
interface IntegerMath {
    int operation(int a, int b);
}

    Method reference

    A metódus hivatkozás egy speciális típusa a lambda kifejezéseknek. Gyakran hozunk létre lambda kifejezést, hogy meghívjunk egy már létező metódust. Az ilyen egyszerű lambda kifejezések kiváltására hozták létre a metódus hivatkozásokat.

    Négy lehetséges típusa van a metódus hivatkozásoknak:

    • Statikus metódus
    • Adott objektum metódusa
    • Tetszőleges objektum bizonyos metódusa
    • Konstruktor

    Hivatkozás egy statikus metódusra

    Lehetőségünk van a lambda kifejezést kiváltani egy olyan statikus metódussal, aminek a paraméterlistája megegyezik a lambda kifejezés paraméterlistájával. Vegyük az alábbi példát, ami kiírja a String lista tartalmát a konzolra és alakítsuk át, hogy method reference segítségével oldjuk meg a feladatot.

    public class StaticMethod {

public static void main (String... args){
    List<String> messages = Arrays.asList("hello", "backend", "csoport!");
    messages.forEach(word -> System.out.println(word));
    }
}

    public class StaticMethod {

public static void main (String... args){
    List<String> messages = Arrays.asList("hello", "backend", "csoport!");
    messages.forEach(System.out::println);
    }
}

    Hivatkozás egy objektum metódusára

    Vegyük az alábbi Bicycle osztályt és a hozzá készített BicycleComparator osztályt, amivel két bicikli példányt tudunk összehasonlítani.

    public class Bicycle {

    private String brand;
    private Integer frameSize;
    // standard constructor, getters and setters
}

public class BicycleComparator implements Comparator {

    @Override
    public int compare(Bicycle a, Bicycle b) {
        return a.getFrameSize().compareTo(b.getFrameSize());
    }

}

    public class Bicycle {
    public static void main(String[] args) {
        BicycleComparator bikeFrameSizeComparator = new BicycleComparator();
        List<Bicycle> bicycleList = Arrays.asList(
                new Bicycle("MTK", 28),
                new Bicycle("TREK", 26),
                new Bicycle("FUJI", 28),
                new Bicycle("KONA", 26)
        );

        bicycleList.stream().sorted((a, b) -> bikeFrameSizeComparator.compare(a, b));
    }
}

    public class Bicycle {
    public static void main(String[] args) {
        BicycleComparator bikeFrameSizeComparator = new BicycleComparator();
        List<Bicycle> bicycleList = Arrays.asList(
                new Bicycle("MTK", 28),
                new Bicycle("TREK", 26),
                new Bicycle("FUJI", 28),
                new Bicycle("KONA", 26)
        );

        bicycleList.stream().sorted(bikeFrameSizeComparator::compare);
    }
}

    Hivatkozás egy objektum metódusára, anélkül, hogy példányosítanánk azt

    Ez a metódushivatkozás nagyon hasonlít az előzre, de ott létre kellett hoznunk, egy BicycleComparator példányt, hogy használni tudjuk. Itt az Integer osztály metódusát tudjuk használni.

    public class MethodReference {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(5, 3, 50, 24, 40, 2, 9, 18);
        numbers.stream().sorted((a, b) -> a.compareTo(b));
    }
}

    public class MethodReference {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(5, 3, 50, 24, 40, 2, 9, 18);
        numbers.stream().sorted(Integer::compareTo);
    }
}

    Hivatkozás egy objektum konstruktorára

    Egészítsük ki a korábbi Bicycle osztályt egy konstruktorral ami egy brand paramétert fogad. Ha létrehozunk egy listát ami a gyártókat tartalmazza egy egyszerű kifejezéssel ebből a String listából Bicycle objektumokat tudunk létrehozni.

    class Bicycle {

    //...
    public Bicycle(String brand) {
        this.brand = brand;
        this.frameSize = 0;
    }
    //...
}

        public class Bicycle {
        public static void main(String[] args) {
            List<String> bikeBrands = Arrays.asList("Giant", "Scott", "Trek", "GT");
            bikeBrands.stream().map(brand -> new Bicycle(brand)).toList();
    }
}

        public class Bicycle {
        public static void main(String[] args) {
            List<String> bikeBrands = Arrays.asList("Giant", "Scott", "Trek", "GT");
            bikeBrands.stream().map(Bicycle::new).toList();
    }
}

    Streams

    Az egyik leghasznosabb újdonság a Java 8 ban a Stream-ek voltak. Nem összekeverendő a Java I/O streamekkel (FileInputStream, stb).

    Egyszerűen fogalmazva a streamek, becsomagolják az adat forrásainkat (List, Array, stb) lehetővé teszik, hogy különböző műveleteket hajtsunk végre az adatokon.

    A stream nem tárol adatot, nem egy adatstruktúra és soha nem módosítja az eredeti adat forrást.

    Stream műveletek

            Employee[] arrayOfEmployees = {
            new Employee(1, "Jeff Bezos", 100000),
            new Employee(2, "Bill Gates", 200000),
            new Employee(3, "Mark Zuckerberg", 300000)
        };

        Stream<Employee> employeeStream = Stream.of(arrayOfEmployees);

    Optional

    A Java 8 egy másik érdekes új funkciója az Optional osztály. Az Optional egy csomagoló osztály, amivel a NullPointerException-ket tudjuk kezelni.
