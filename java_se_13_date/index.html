    Date

    Bevezetés

    A Java 8 új API-kat vezetett be a Date és Time funkciókhoz, hogy kiküszöbölje a régebbi java.util.Date és java.util.Calendar hiányosságait. Ebben a szekcióban megismedünk az új dátum kezelő osztályokkal és megnézzük milyen hiányosságokat oldott meg az új API.

    Problémák a régebbi Dátum API-val

    • Szál biztonság – A Date és Calendar osztályok nem voltak szálbiztosak, így a fejlesztőkre hárult a feladat, hogy megoldják a konkurenciából fakadó gondokat. Az új keretrendszerben a dátum kezelő osztályok már immutable objektumok, vagyis a létrehozás után nem lehet módosítani az értéküket és szálbiztosak, így a konkurencia feloldására már nem kell a fejlesztőknek saját megoldásokat implementálniuk.
    • Rosszul megtervezett API – A korábbi API rosszul volt megtervezve és nem támogattak megfelelő metódusokat a dátumok megfelelő használatához. Az új Date/Time API ISO-központú, és konzisztens modelleket használ a dátum, idő, időtartam és időszakok használathoz. A legáltalánosabb műveleteket támogató segédmetódusok széles skálája segíti a dátumok használatát.
    • Időzónák – A régebbi dátum modellek nem támogattak időzónákat így a fejlesztőknek kellett saját megoldásokat implementálni, ha időzónákat szerettek volna kezelni. A Java 8 -ban behozott új dátum osztályok már képesek különböző időzónákat kezelni.

    LocalDate, LocalTime és LocalDateTime

    A leggyakrabban használt osztályok a LocalDate, LocalTime és LocalDateTime. Ahogy a nevük is mutatja, a helyi dátumot/időt reprezentálják a használó kontexusában. Ezeket az osztályokat főleg akkor használjuk, ha az időzónákat nem kell kifejezetten megadni.

    LocalDate

    A LocalDate egy dátumot reprezentál ISO formátumban (éééé-hh-nn) idő nélkül. Használhatjuk olyan dátumok tárolására, mint például a születésnapok.

            LocalDate localDate=LocalDate.now(); // (1)
        localDate=LocalDate.of(2015,2,20); // (2)
        localDate=LocalDate.parse("2015-02-20"); // (3)
    1. Létrehozhatunk LocalDate objektumot az aktuális dátummal
    2. Használhatjuk a konstruktorát
    3. Előállíthatjuk egy String-ből is

    A LocalDate osztály számos extra metódust tartalmaz amivel módosítani tudjuk a dátum objektumot.

            LocalDate tomorrow=LocalDate.now().plusDays(1); // (1)

        LocalDate previousMonthSameDay=LocalDate.now().minus(1,ChronoUnit.MONTHS); // (2)
    1. A meglévő dátumhoz hozzáadhatunk egy napot
    2. Vagy kivonhatunk egy hónapot

    További dátum manupulációs metódusok

            DayOfWeek today=LocalDate.now().getDayOfWeek();
        int todayInt=LocalDate.now().getDayOfMonth();

        boolean leapYear=LocalDate.now().isLeapYear();

        boolean notBefore=LocalDate.parse("2016-06-12")
        .isBefore(LocalDate.parse("2016-06-11"));

        boolean isAfter=LocalDate.parse("2016-06-12")
        .isAfter(LocalDate.parse("2016-06-11"));

    LocalTime

    A LocalTime dátum nélküli időpontot reprezentál.

            LocalTime now=LocalTime.now(); // (1)
        now=LocalTime.of(6,30); // (2)
        now=LocalTime.parse("06:30"); // (3)
    1. Létrehozhatunk LocalTime objektumot az aktuális időponttal
    2. Használhatjuk a konstruktorát
    3. Vagy előállíthatjuk egy String-ből is

    LocalDateTime

    A LocalDateTime a dátum és időpont kombinációját tartalmazza.

            LocalDateTime localDateTime=LocalDateTime.now();  // (1)
        LocalDateTime.of(2015,Month.FEBRUARY,20,06,30);  // (2)
        LocalDateTime.parse("2015-02-20T06:30:00");  // (3)
    1. Létrehozhatunk LocalTime objektumot az aktuális dátummal és idővel
    2. Használhatjuk a LocalDateTime
    3. Vagy előállíthatjuk egy String-ből is

    ZonedDateTime API

    A Java 8 biztosítja a ZonedDateTime-ot, amikor az időzóna-specifikus dátummal és időponttal kell foglalkoznunk. A ZoneId egy azonosító, amely különböző zónákat jelöl. Körülbelül 40 különböző időzóna létezik, és a ZoneId ezeket a következőképpen ábrázolja.

    A lehetséges időzónákat az alábbi metódussal kaphatjuk meg:

            Set<String> allZoneIds=ZoneId.getAvailableZoneIds();

    Vagy elkérhetjük egy időzóna azonosítóját a következő metódussal:

            ZoneId zoneId=ZoneId.of("Europe/Paris");

    Maga a ZonedDateTime-t pedig egy LocalDateTime és ZoneId felhasználásával tudjuk létrehozni vagy feldolgozhatunk egy Stringet is.

            ZonedDateTime zonedDateTime=ZonedDateTime.of(localDateTime,zoneId);
        ZonedDateTime.parse("2015-05-03T10:15:30+01:00[Europe/Paris]");

    Period és Duration

    A Period osztály időmennyiségét tartalmaz években, hónapokban és napokban kifejezve, a Duration osztály pedig másodpercben és nanoszekundumban kifejezve.

    A Period osztályt arra használjuk, hogy módosítsunk egy meglévő dátumon vagy a különbség kiszámítására két dátum között.

            LocalDate initialDate=LocalDate.parse("2007-05-10");

    Módosíthatjuk a dátumot a Period segítségével.

            LocalDate finalDate=initialDate.plus(Period.ofDays(5));

    A Period osztály számos lekérdező metódust tartalmaz a különböző időegységek lekéréshez, például: getYears , getMonths és getDays.

            int daysBetween=Period.between(initialDate,finalDate).getDays();

    A Period osztályhoz haszonlóan a Duration osztályt használjuk, amikor Time objektumokkal dolgozunk.

    Hozzunk létre egy LocalTime objektumot, majd készítsünk egy másikat, ami 30 másodperccel későbbi időpontot tartalmaz.

            LocalTime initialTime=LocalTime.of(6,30,0);
        LocalTime finalTime=initialTime.plus(Duration.ofSeconds(30));

    Majd a Duration osztály felhasználásával megnézhetjük a különbséget másodpercben a két Time objektumunk között.

            long differenceInSeconds=Duration.between(initialTime,finalTime).getSeconds();
